# Workflow d’interaction entre agents IA – CCCZ

Ce document définit le **workflow opérationnel officiel** d’interaction entre les agents IA du projet numérique du **Centre Culturel Congolais Le Zoo (CCCZ)**.

Il précise **qui intervient**, **à quel moment**, **avec quels livrables** et **sous quelle validation**.

---

## 🧠 Principe Général de Fonctionnement

- **L’Agent Manager / Chef de Projet IA** est le point central.
- Aucun agent ne travaille sans mission claire.
- Chaque phase se termine par une validation avant passage à la suivante.
- Le workflow respecte la hiérarchie institutionnelle et la mission culturelle du CCCZ.

---

## 🔁 Vue Globale du Workflow

```
Direction Générale
        ↓
Agent Manager IA
        ↓
Agent Product Owner
        ↓
UX/UI Designer
        ↓
Frontend ↔ Backend ↔ DevOps
        ↓
Content & Communication
        ↓
Agent QA
        ↓
Validation finale & mise en ligne
```

---

## 🧩 Phase 1 – Cadrage Stratégique

### Agents impliqués
- Agent Manager IA (leader)
- Agent Product Owner

### Actions
- Analyse du Plan d’Action 2024–2026
- Définition des objectifs numériques
- Identification des priorités culturelles

### Livrables
- Vision produit
- Backlog initial
- Définition du MVP

### Validation
- Agent Manager
- Direction Générale (si requis)

---

## 🎯 Phase 2 – Définition Fonctionnelle

### Agents impliqués
- Agent Product Owner (leader)
- Agent Manager

### Actions
- Rédaction des user stories
- Priorisation des fonctionnalités
- Découpage par phases

### Livrables
- Backlog priorisé
- Roadmap fonctionnelle

### Validation
- Agent Manager

---

## 🎨 Phase 3 – Design UX/UI

### Agents impliqués
- Agent UX/UI Designer (leader)
- Agent Product Owner
- Agent Manager

### Actions
- Conception des parcours utilisateurs
- Design des interfaces clés
- Application de la charte graphique CCCZ

### Livrables
- Wireframes
- Maquettes UI
- Design system

### Validation
- Agent Manager
- Product Owner

---

## 💻 Phase 4 – Développement Technique

### Agents impliqués
- Frontend Developer
- Backend Developer
- DevOps

### Actions
- Implémentation frontend
- Développement des API et données
- Mise en place de l’infrastructure

### Règles d’interaction
- Frontend ↔ Backend : API contractuelles
- DevOps supporte CI/CD et sécurité
- Tout changement majeur est validé par l’Agent Manager

### Livrables
- Code source
- API documentées
- Environnements fonctionnels

---

## 📚 Phase 5 – Contenu & Culture

### Agents impliqués
- Agent Content & Cultural Curator (leader)
- Agent Communication

### Actions
- Structuration des contenus culturels
- Rédaction et intégration
- Mise en valeur des artistes et événements

### Livrables
- Fiches artistes
- Contenus culturels
- Archives numériques

### Validation
- Agent Manager
- Directions concernées (Bibliothèque, DACPA)

---

## 📢 Phase 6 – Communication & Diffusion

### Agents impliqués
- Agent Communication (leader)
- Agent Manager

### Actions
- Intégration réseaux sociaux
- Mise en avant des activités
- Stratégie de visibilité

### Livrables
- Plan de communication
- Pages partenaires
- Indicateurs d’engagement

---

## 🛡️ Phase 7 – Qualité & Conformité

### Agents impliqués
- Agent QA (leader)
- Tous les agents concernés

### Actions
- Tests fonctionnels
- Vérification conformité institutionnelle
- Tests sécurité et accessibilité

### Livrables
- Rapport QA
- Liste des corrections

### Validation
- Agent Manager

---

## 🚀 Phase 8 – Déploiement & Évolution

### Agents impliqués
- DevOps
- Agent Manager

### Actions
- Mise en production
- Suivi post-déploiement
- Collecte retours utilisateurs

### Livrables
- Version en ligne
- Plan d’amélioration continue

---

## ⚖️ Règles d’Or du Workflow

- Toute décision stratégique passe par l’Agent Manager
- Toute fonctionnalité doit servir la mission culturelle
- La simplicité prime sur la complexité
- Le patrimoine numérique est prioritaire
